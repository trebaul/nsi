    I. Représenter le texte en machine¶

    Cours¶

    A. Le code ASCII¶

    Le code ASCII (American Standard Code for Information Interchange) a été proposé dans les années 60 pour standardiser la représentation des caractères. Les mots sont stockés sur 1 octet, et seul les 128 premières valeurs sont utilisées.

    Combien de bits sont utilisés ?

    Pour coder 128 valeurs (de 0 à 127), il faut 7 bits car \(2^7=128\). Il reste donc 1 bit non-utilisé sur cet octet.

    Il ne permet que de représenter les caractères principaux de la langue anglaise, ainsi que des caractères non-imprimables (comme le retour à la ligne). Il pose un problème pour représenter certains caractères du français, qui n'existent pas dans la langue anglaise.

    Quels caractères ne sont pas représentables ?

    Les caractères accentués, ou bien le "ç" par exemple posent problème.

    En programmation, on peut avoir l'équivalence avec les fonctions ord et chr : 

    >>> ord("k")
107
>>> chr(52)
'4'

    B. La norme ISO 8859-1¶

    Pour pallier ce problème, la norme ISO 8859-1, aussi appelée Latin-1 a vu le jour pour représenter tous les caractères de l'alphabet latin (au nombre de 191) en utilisant 8 bits.

    Pourquoi est-ce suffisant ?

    Sur 8 bits, on peut représenter $2^8=256 caractères (de 0 à 255). C'est donc suffisant pour représenter 191 caractères.

    Cette norme permet donc de représenter les caractères utilisés dans la plupart des langues européennes. Mais elle laisse de côté d'autres alphabets, comme le cyrillique, ou bien les idéogrammes utilisés en chinois ou en japonais.

    C. Unicode¶

    En 1991 voit le jour la norme qui est utilisée aujourd'hui par plus de 95% des sites web : Unicode. Comme son nom l'indique, elle a pour vocation d'intégrer tous les caractères existants, pour pouvoir lire n'importe quel texte sans avoir à changer la configuration de son traitement de texte.

    C'est une table qui répertorie tous les codages existants : elle ne crée pas un nouveau codage. Elle intègre donc notamment le code ASCII. Les systèmes de codage sont UTF-8 (utilisée sur le Web), UTF-16, UTF-32 (des variations, utilisant plus de bits).

    Pour UTF-8, les caractères sont encodés avec un nombre variable d'octets :
    - les caractères les plus "classiques" sont codés sur 1 octet (ils correspondent au code ASCII),
    - les caractères moins classiques peuvent être codés sur jusqu'à 4 octets.
