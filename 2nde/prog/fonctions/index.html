    III. Les fonctions¶

    Cours¶

    A. Exemple¶

    On veut créer un programme qui vérifie si un utilisateur a entré le bon identifiant et mot de passe associé : on parle de bonne authentification sur un réseau.

    On utilise les variables identifiant et motDePasse pour stocker les informations entrées par l'utilisateur. L'ordinateur attend un identifiant et un mot de passe précis pour confirmer la connexion : on utilise une expression qui combine deux conditions avec le mot-clé and.

    A.I. Un premier programme¶

    On donne le code suivant (incomplet) :

    identifiant = "eleveSNT"
motDePasse = "mot"

if .............................. and motDePasse == "Mprx24$":
    autorisee = True
else:
    ....................................
    1. Relever les variables utilisées, et indiquer leur type.
    • identifiant et motDePasse sont de type chaîne de caractères,
    • autorisee est de type booléen.
    2. Compléter ensuite ce programme avec la condition vérifiant que l'identifiant est le bon et la bonne valeur stockée dans la variable contenant le résultat de l'algorithme : autorisee. 
    identifiant = "eleveSNT"
motDePasse = "mot"

if identifiant == "eleveSNT" and motDePasse == "Mprx24$":
    autorisee = True
else:
    autorisee = False
    3. Que va contenir la variable autorisee, après exécution du code ?

    Elle va contenir False, car le mot de passe entré n'est pas le bon.

    4. Que faut-il faire pour tester une autre combinaison d'identifiant et de mot de passe ?

    Il faut : - soit rééxécuter le programme en changeant les valeurs stockées dans les variables, - soit réécrire exactement le même code une deuxième fois.

    5. Quelle est la principale différence entre les variables identifiant et motDePasse et celui de autorisee ?

    identifiant et motDePasse sont des variables que l'utilisateur(le programmeur) modifie, alors que la valeur de autorisee est calculée automatiquement par l'ordinateur.

    A.II. Avec une fonction¶

    Nous allons voir un outil très puissant permettant de faire plusieurs tests d'affilée, de tester plusieurs fois le même code en ne changeant que des valeurs contenues dans des variables : les fonctions.

    def testeConnexion(identifiant, motDePasse):
    if .............................. and motDePasse == "Mprx24$":
        autorisee = True
    else:
        ....................................
    return autorisee
    1. Compléter le code de la fonction testeConnexion, comme fait dans la partie précédente. 
    def testeConnexion(identifiant, motDePasse):
    if identifiant == "eleveSNT" and motDePasse == "Mprx24$":
        autorisee = True
    else:
        autorisee = False
    return autorisee
    2. Que remarque-t-on pour les variables identifiant et motDePasse ?

    Ces variables ne sont plus initialisées avant le programme, mais écrites entre les parenthèses après le nom de la fonction.

    3. Que remarque-t-on pour la variable autorisee ?

    Sa valeur est affectée de la même façon, mais elle est en plus placée après le mot-clé return.

    4. Compléter la signification des mots-clés suivant :
    • def indique que l'on définit une fonction, dont on donne le nom et les paramètres.
    • return indique les valeurs que l'on veut renvoyer à l'utilisateur, le résultat du traitement fait par la fonction.
    5. Pour tester plusieurs combinaisons d'identifiants et mots de passe, on fait des appels de la fonction testeConnexion. 
    resultat1 = testeConnexion("eleveSNT", "mot")
resultat2 = testeConnexion("SNT", "Mprx24$")
resultat3 = testeConnexion("eleveSNT", "Mprx24$")

    B. Cas général¶

    Une fonction est un sous-programme qui peut être utilisé plusieurs fois dans le programme principal. Ce sous-programme effectue un traitement (une suite d'instructions) sur des données en paramètres.

    On la définit de la manière suivante : 

    def nom_de_la_fonction(param1, param2,...)
    {instructions}
    return resultat

    On peut le représenter schématiquement de la façon suivante.

    graph LR
param1 --> F[nom_de_la_fonction];
param2 --> F[nom_de_la_fonction];
F --> resultat

    On appelle la fonction soit en donnant directement des valeurs aux paramètres, soit en stockant ces valeurs dans des variables.

    Donner un exemple pour les deux méthodes. 
    #méthode 1
r = nom_de_la_fonction("bonjour", 2)
#méthode 2
a = "bonjour"
b = 2
r = nom_de_la_fonction(a, b)
