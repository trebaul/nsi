    I. Le fonctionnement du Web¶

    Cours : Les bases du Web¶

    Introduction¶

    Qu’est-ce que le Web ?

    Le web est une application d'Internet : une manière d'utiliser Internet. Elle permet d'accéder à des pages Web.

    Quels différents contenus peut-on trouver sur une page Web ?

    Une page web peut contenir du texte, des images, sons, vidéos, et des liens vers d'autres pages Web.

    Quel logiciel est-ce qu'on utilise pour se connecter sur le Web ? En donner des exemples.

    On utilise un navigateur, comme Mozilla Firefox, Google Chrome, ou bien Safari.

    I. URL¶

    On accède à une page Web grâce à son URL (Uniform Ressource Locator). Cette adresse localise la page, grâce à sa structure.

    Exemple

    https://pixees.fr/informatiquelycee/prem/index.html est constituée de :
    - "https" : protocole de communication, définit comment accéder à la page
    - "pixees.fr" : nom de domaine du serveur, l'identifie grâce à son lien avec son adresse IP
    - "/informatiquelycee/prem/index.html" : chemin d'accès à la page sur le serveur

    II. Les langages du Web¶

    La page Web est un fichier au format html, le langage de base du Web. Le HTML (Hypertext Markup Language) définit la structure et le contenu de la page.

    Qu'est-ce qui est alors modifiable avec le HTML ?

    On peut modifier les contenus des paragraphes, des titres, et des contenus multimédia.

    Une page Web est constituée au minimum de HTML. Elle peut être écrite avec de CSS pour sa mise en forme et des langages de programmation comme le Javascript pour être intéractive.

    III. Le principe de l'hypertexte¶

    On navigue sur le Web en cliquant de lien en lien, appelés liens hypertextes. C'est de ce principe que vient son nom, "la toile".

    Activité 1 : le protocole HTTPS¶

    • A partir du document suivant, répondre aux questions.

      Techniquement, quelle est la différence entre HTTP et HTTPS ?

      Avec HTTPS, les informations sont chiffrées, c’est-à-dire incompréhensible pour quiconque en dehors du client et du serveur.

      Avec le protocole HTTPS, qu’est-ce qui permet au serveur de décrypter les données reçues ?

      Une clé permet au serveur de décrypter les données reçues. Seuls lui et le client la possèdent.

    Dans un navigateur, comment repérer si l’on utilise HTTPS ?

    Le nom du protocole est écrit au début de l’URL, et un cadenas s’affiche avant celle-ci.

    Activité 2 : les cookies¶

    Souvent, lorsque l’on visite un site Web, on reçoit un message indiquant que celui-ci utilise des cookies et nous demande si on les accepte.

    A partir de la vidéo de la CNIL, répondre aux questions : https://youtu.be/a9kiQPV_SPg

    Qu’est-ce qu’un cookie ?

    Un cookie est un fichier stocké par un serveur sur notre ordinateur qui collecte des données de navigation.

    Donner des exemples d’informations utiles stockées par les cookies.

    Stocker le contenu de notre panier (sur un site de e-commerce), pour pouvoir le retrouver lors d’une nouvelle connexion.

    Donner des exemples d’informations stockées servant à faire de la publicité ciblée.

    Les produits regardés, les liens cliqués, le temps passé sur une page. Ces informations peuvent être revendues.

    Cours : Sécurité sur le Web¶

    A compléter.

    Sur le Web, la sécurité est assurée notamment grâce au protocole de communication HTTPS. Il permet le chiffrement des données échangées et est aujourd'hui beaucoup plus utilisé qu'HTTP.

    Des données sont tout de même partagées aux serveurs Web des sites auxquels on accède, sous la forme de cookies. Ces informations peuvent nous être utiles, mais servent aussi à enregistrer des données personnelles pour faire de la publicité ciblée. Il est dans certains cas possible de refuser des cookies non-essentiels.
